# Stochastic Gradient Descent
for iter in range(iterations):
    y_pred = m * x + c
    mse = (1 / num_of_prices) * sum((y - y_pred) ** 2)
    for i in range(num_of_prices):
        if(mse < 0.05):    # stopping condition
            break
        gradient_m = (-2 / num_of_prices) * (x[i] * (y[i] - y_pred[i]))
        gradient_c = (-2 / num_of_prices) * (y[i] - y_pred[i])
        m = m - L * gradient_m
        c = c - L * gradient_c
    
    # Lists for plotting
    mse_values.append(mse)
    gradient_m_values.append(gradient_m)
    gradient_c_values.append(gradient_c)
    m_values.append(m)
    c_values.append(c)

(e) Explain the pros and cons of gradient descent and stochastic gradient descent.¶

Gradient Descent (aka Batch Gradient Descent) is when all of the data is used for each iteration. The pros of this method are that it is accurate and unbiased. It can also theoretically converge to a global optimum for convex functions and local optimum for other functions. However, since all the data is used for each iteration, if there is a lot of data, it can be very slow. It is also easy for it to get trapped into local modes. A way to solve these issues is with Stochastic Gradient Descent.

Stochastic Gradient Descent is when a single sample of the data is used for each update step. This method is faster as a result. It also adds noise to the learning process which can make the model more general. However, the noise can also be a problem, as it can cause the learning to go back and forth. Also, it can be quite slow to converge to a solution.
